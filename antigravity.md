# Antigravity Analysis: WorkSphere Enterprise-Grade Activity Intelligence

## 1. Project Overview
**WorkSphere** is a massive, enterprise-grade Activity Intelligence and Monitoring Platform tailored for large Multinational Corporations (MNCs) with over 5,000 employees. It features an incredibly complex architecture integrating AI services, microservices, a monolithic core backend, a robust frontend, and mobile capabilities.

## 2. Architecture & Directory Structure

The repository is modularized into distinct service layers within `apps/enterprise-monitoring-system`:

### 2.1 AI Services (`/ai-services`)
A Python-based suite of machine learning models and API routers designed for advanced analytics:
- **Anomaly Detection**: Flags unusual employee activities or data breaches.
- **Behavior Analysis**: Models employee behaviors and identifies trends.
- **Face Recognition**: Security and attendance tracking.
- **Productivity Prediction**: ML models to forecast burnout or peak productivity periods.
- **Recommendation Engine**: Suggests task delegations and team formations.
- **Report AI**: Automated narrative generation from telemetry data.
- **Sentiment Analysis**: Evaluates communication and feedback sentiment.
- **Tracking Analysis**: Deep dives into telemetry data (mouse, keyboard, active windows).
- **Violence Detection**: Security video feed analysis.

### 2.2 Microservices (`/microservices`)
A distributed system composed of specialized services, likely communicating via Kafka/Redis and orchestrated by a Gateway:
- **auth-service**: Authentication, authorization, and RBAC handling.
- **employee-service**: Employee lifecycle and profile management.
- **hr-service**: Human Resources features, leave approvals, and recruitment pipelines.
- **monitoring-service**: Ingests high-throughput telemetry from agents.
- **gps-service**: Location tracking for mobile/remote workers.
- **analytics-service**: Data aggregation and metric generation.
- **report-service**: Document generation and auditing.
- **notification-service**: Real-time alerts, emails, and pushes.
- **gateway-service**: API Gateway routing traffic to microservices.

### 2.3 Core Backend (`/backend`)
A monolithic Node.js/Express (with Java elements for Enterprise layers like CTO/CEO dashboards) and Prisma ORM for PostgreSQL. This serves as the primary data orchestrator for the system's core entities.
- Interfaces directly with the Database.
- Houses controllers, services, and DTOs (e.g., `CeoController`, `CtoService`).

### 2.4 Frontend Applications
- **`/frontend`**: The primary web application (React, TypeScript, Vite, TailwindCSS). It encompasses distinct modules for 18 Enterprise Roles, including:
  - CEO Executive Dashboard (Financials, Strategic Planning).
  - CTO Command Center (CloudOps, Infrastructure, Engineering Analytics).
  - HR Executive Dashboard (Budgeting, Resource Allocation, Leave Approvals).
  - Modules for QA, DevOps, Marketing, and Software Engineers.
  - **Design System**: Recently refactored to a premium Neumorphic/Skeuomorphic 3D aesthetic (`.skeuo-panel`, `.skeuo-inset`, `.skeuo-btn`).
- **`/web-page`**: Landing page or public-facing portal for the product.

### 2.5 Other Key Components
- **`/mobile-app`**: React Native/Flutter app for employee tracking and mobile dashboards.
- **`/database` & `/docker` & `/deployment`**: Infrastructure as Code (IaC) for Postgres, Redis, Kafka, and containerization.
- **`/nginx` & `/gateway`**: Reverse proxies and load balancers ensuring high availability.
- **`/websocket-server`**: Dedicated real-time communication server (Socket.io) for live tracking, chat, and instant telemetry updates.

## 3. Technology Stack Summary
- **Frontend**: React 18, TypeScript, Vite, TailwindCSS 4, Framer Motion, Recharts, Redux.
- **Backend Core**: Java Spring Boot (Enterprise Layers), Node.js/Express, Prisma ORM.
- **Microservices**: Node.js, Python (AI).
- **Database Layer**: PostgreSQL (Persistent), Redis (Caching), Kafka (Message Queue).
- **AI/ML**: Python, TensorFlow/PyTorch, FastAPI (for routers).
- **Infrastructure**: Docker, Nginx.

## 4. Security & RBAC
The platform implements a strict 18-tier Role-Based Access Control (RBAC) matrix ranging from `SUPER_ADMIN` and `CEO` down to `INTERN`. Data isolation ensures that telemetry and strategic insights are only available to authorized personnel, preventing data leaks in a 5,000+ employee environment.
